## Schedule Correction

The initial preflight manifest used the default `ref_consistency_start=3000`, which would not activate the RC loss in a 300-iteration validation run because `train.py` gates it with `iteration >= opt.ref_consistency_start`.

The corrected i300 validation manifest uses:

- `ref_consistency_start=60`
- `ref_consistency_every=4`
- `roughness_smoothness_start=60`

This keeps the run short while ensuring the `rc` variant exercises the intended reflection-consistency training branch before iteration 300.
